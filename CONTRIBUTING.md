## Getting Started

1. Fork the repository
2. See [how-to-run.md](./how-to-run.md) for local setup
3. See [docs/development.md](./docs/development.md) for full guide

## Code Style

- TypeScript strict mode
- ESLint 9 flat config — run `npm run lint` before committing
- Components: kebab-case filenames
- Server Components by default, `"use client"` only when needed
- TailwindCSS utility classes for styling

## Branching

- `main` — production-ready
- Feature branches: `feat/<description>`
- Fix branches: `fix/<description>`

## Commit Messages

Use clear, concise messages describing what and why.

## Pull Requests

- Link related issues
- Describe changes in detail
- Ensure lint passes
- Update docs if adding/changing routes or components
